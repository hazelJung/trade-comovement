## 📈 주요 발견사항

### 1. 기계산업 중심 데이터셋
- 전체 무역량의 **85.8%**가 기계산업 관련
- 품목의 **67%**가 기계산업 관련
- 이는 우연이 아닌 **의도적인 기계산업 공급망 데이터**로 보임

### 2. 공급망 계층 구조 명확
- 원자재 → 중간재 → 최종제품의 3단계 구조가 뚜렷함
- 각 단계별로 상관관계가 존재할 가능성 높음

### 3. 핵심 품목 식별
- **전자석 (8505)**: 가장 큰 무역량 (1.57조)
- **화학제품 (3824)**: 두 번째 큰 무역량 (4.75조)
- **전기모터 (8501)**: 5개 품목, 9.2천억 무역량

### 4. 배터리 산업 연관성
- 니켈 (2805, 8102)과 코발트 (8105)는 전기차 배터리 원자재
- 전자석 (8505)은 전기모터의 핵심 부품
- 이는 **전기차/배터리 산업 공급망**을 포함하고 있음을 시사

## 🎯 결론

### train.csv는 **기계산업 공급망 데이터**입니다.

1. **높은 관련성**: 전체의 85.8%가 기계산업 관련
2. **명확한 구조**: 원자재 → 중간재 → 최종제품 계층 구조
3. **실제 공급망 반영**: HS 코드 분류가 실제 산업 공급망과 일치
4. **배터리/전기차 산업 포함**: 니켈, 코발트, 전자석 등

### 현재 코드의 문제점
현재 `train_model.py`는 이러한 **도메인 지식을 전혀 활용하지 않고 있습니다**:
- HS4 코드를 사용하지 않음
- 공급망 계층 구조를 고려하지 않음
- 원자재-중간재-최종제품 관계를 무시

### 개선 제안
1. HS4 코드를 feature로 추가
2. 공급망 계층별로 그룹화하여 분석
3. 같은 HS4 그룹 내에서 우선 공행성 탐색
4. 공급망 방향성 고려 (원자재 → 중간재 → 최종제품)
